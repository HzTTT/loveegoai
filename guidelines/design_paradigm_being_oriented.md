# Love Ego AI — BEING 导向设计范式

> 基于 @Aliec 2026-04-13 重新设计方案，整合 @codex 产品逻辑修正 + @claude-sonnet 技术映射
> 状态：待 @ELENA 最终确认

---

## 核心转变

**从**：多邻国式闯关系统（答题、得分、连胜、进度条）= DOING 机制
**到**：每日身体练习的陪伴工具 = BEING 容器

**核心定位**："不是让你学会更多，而是让你重新回到自己。"（@codex 定稿）

---

## 核心体验循环（每次打开 App）

```
打开 App
  → "你今天感觉怎么样？"（ELENA 每节课的开场）
  → 身体签到：身体轮廓图，点击有感觉的部位
  → 选择练习（或 AI 根据时间/状态建议）
  → 引导式练习（音频 + 极简视觉，3-10 分钟）
  → 练习后反思：再次身体签到 + 自由文字记录
  → 结束（肯定语 / 或只是安静地关闭）
```

---

## 五大设计原则

### 1. 打开，不关闭
- 多邻国：完成任务 → 关闭循环 → 奖励 → 下一个任务
- 我们：打开练习 → 感受身体 → 留在感受里 → 没有"完成"徽章
- 练习结束的标志不是"答对了"，而是用户觉得准备好了

### 2. 日记，不考试
- 练习后提示："你现在身体哪里有感受？是扩张的还是收缩的？"
- 输入方式：身体轮廓点击 + 柔性三档（收缩/中性/扩张）+ 自由文字
- **不做硬百分比，不把 BEING 变成 KPI**（@codex）

### 3. 节奏，不连胜
- 没有"连胜 X 天"的压力机制
- 温和邀请（"现在想做一个呼吸练习吗？"）
- 跳过几天回来：没有惩罚 UI，只有"欢迎回来"
- 练习记录用日历小点呈现，不是进度条

### 4. AI 教练是陪伴，不是老师
- 默认：先引导回到身体（"先来三次垂直呼吸"）
- 分流逻辑（@codex）：
  - 情绪激动 → 先接住（"我听到你了"）→ 再引导到身体感受
  - 产品问题 → 直接回答
  - 练习复盘 → 从感受切入，不重复引导
- 永远不替用户"分析"情绪，而是帮用户在身体里定位感受
- 关键问句：**"这个感受在你身体的哪个位置？是什么质感？"**

### 5. 视觉语言：流动，不结构化
- 不用硬直线、方格、进度条
- 用有机的、呼吸感的视觉（水流、光晕、呼吸节奏动画）
- 色温随状态变化（练习时温暖柔和，日记时清凉安静）
- 整体氛围：安全、柔软、允许你什么都不做

## MVP 核心练习单元（来自 ELENA 课程）

| 练习 | 时长 | 核心 |
|---|---|---|
| 垂直呼吸 | 3 min | 电梯比喻，能量从头顶下沉到海底轮 |
| Body Scan | 5 min | 扫描身体，感知当下能量状态 |
| 感官练习 | 5 min | 闭眼激活五感（视/嗅/听/触/体感）|
| 海底轮激活 | 5 min | 盘腿坐 + hip rotation + 肯定语 |
| Goddess Rising Ritual | 10 min | 晨间仪式：呼吸→胸部按摩→海底轮→肯定语 |

先把这 5 个做顺，不做大而全课程平台。

## 四阶段隐性进阶（内部框架，不贴用户标签）

- **内部用**：根据用户练习频率、日记深度、身体感知变化来判断阶段
- **外部呈现**：不显示等级，只在 AI 对话中体现变化感
  - 早期："试试感受一下你的呼吸温度"
  - 中期："你上次提到胸口有紧绷感，现在呢？"
  - 后期：更少引导，更多空间（因为用户已经能自己回到身体）

## 技术实现映射（@claude-sonnet）

- **身体签到数据模型**：`sensation_location[]`（身体部位）+ `state: "contract|neutral|expand"` + 自由文字，无百分比
- **练习引导**：每个练习 = TTS 音频（edge-tts）+ 计时器 + 呼吸动画，5 个引导页面 + 练习后反思记录
- **AI 教练 intent classification**：产品问题→直接回答 / 情绪激动→接住模式 / 练习复盘→感受跟进 / 默认→引导回身体
- **无 streak 惩罚**：跳过 N 天回来只显示温和欢迎
